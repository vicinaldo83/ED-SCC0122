Tipo Abstrato de Dado (TAD)

Conceito parecido com a idea de classes onde há um encapsulamento de informações e operações específicas com as mesmas

Util para criar tipo específicos de dados para serem re-utilizados dentro de diversos programas.

Ordem de verificação para criação de um TAD:

  • Mundo real: Qual é o objeto prncipal
  • Informação: O que vamos tirar do mundo real
  • Tipo: Tipo de dados efetivamente dentro do computador
  • Operações: Operações definidads para seu topo de dado personalizado

    • Observações:

    Definir uma especificação com foco em desempenho, eficiência ou facilidade.

    Ex.: Fazer um sistema de agenda de endereços

    • Especificações do TAD
      • Dados / Informações
      • Operações
    • Implementação

    Para um agenda, seria necessário as seguintes informações: Id, Nome da pessoa, *telefone.

    Operações básicas: Adicionar pessoa, remover pessoas, query ID, query Nome, query Telefone (talvez para uma pesquisa por estado, número fixos ou celular, etc), update Numero

    Nome do dado Tipo do dado
    Nome da pessoa String
    Id da pessoas Long
    Telefone DDD - Int[2] / Numero - Int[9]
    Operação Descrição Parâmetros Retorno
    Adicionar pessoa Adiciona uma pessoa a agenda id boolean
    Remover pessoa Remove a pessoa da agenda id boolean
    Editar número Editar número de telefone da pessoa id, novo número (Number) boolean
    Editar endereços Editar endereço da pessoa passada id, novo endereço boolean
    Pesquisar Procura na agenda as pessoas referentes as informações passadas nome ou endereço Array [id]
    Pesquisar número Pesquisar por número de telefone ddd ou número Array [id]
